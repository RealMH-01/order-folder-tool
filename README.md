# 订单文件夹自动创建工具

一款专为业务员 / 跟单员设计的 Windows 桌面工具，用于一键生成标准订单目录结构、
自动复制模板文件并重命名、自动生成文件清单 Excel。支持外贸 / 内贸、单笔 / 批量、
多业务员 / 多客户 / 多模板管理。

- 开发语言：Python 3.9+ / PyQt5
- 依赖库：PyQt5、openpyxl、pyinstaller
- 适用平台：Windows（同样可在 Linux / macOS 以源码方式运行）

---

## 一、功能一览

| 模块 | 功能 |
| --- | --- |
| 启动页 | 设置"公司资料根目录"、"模板文件目录"、一键扫描导入业务员 |
| 单笔创建 | 选择业务员 / 客户 / 订单类型 / 模板 / 产品类别 → 填表 → 预览 → 创建 |
| 批量导入 | 支持 Excel 导入 / 手动添加多行一次创建 |
| 模板管理 | 新建 / 编辑 / 另存为个人或客户模板 / 删除 / 预览 |
| 历史记录 | 记录每次创建结果，支持搜索过滤 |
| 使用帮助 | 内嵌傻瓜式图文教程，支持章节快速跳转 |

创建时自动完成：
1. 按模板创建所有文件夹（已存在的会自动跳过，缺失的会补建）。
2. 把模板文件目录中的模板文件复制到目标位置，并按规则重命名。
3. 在订单号文件夹内生成 `文件清单-<订单号>.xlsx`，可手动打勾。
4. 写入操作历史。

### 订单文件夹生成位置

```
<根目录>/1订单/<业务员rel_path>/[<mid_layer>/]<客户名>/<订单号>/
                                                         ├── SD/
                                                         ├── 报关资料/
                                                         ├── 货代资料/
                                                         ├── ...
                                                         └── 文件清单-<订单号>.xlsx
```

- **业务员 rel_path**：可以是 `张莹莹` 这种直接路径，也可以是 `湖北/文天堂` 这种分公司下的路径。
- **mid_layer**：若业务员文件夹下含有同时包含「进行」和「订单」两个关键词的中间层（如 `进行中订单`、`1.进行订单`），则作为中间层保留；否则为空。
- 具体结构由程序首次启动时自动扫描生成（见下方"扫描导入业务员"）。

## 二、使用方法（面向"从没装过 Python"的小白）

### 第一步：安装 Python

1. 打开浏览器访问 <https://www.python.org/downloads/>
2. 点击 **Download Python 3.x.x**（黄色大按钮）。
3. 下载完成后双击安装包。
4. **⚠ 最关键：在安装向导的第一页，一定要勾选 **`Add Python to PATH`** 这个复选框**，然后点击 **Install Now**。
5. 等安装完成，点击 Close。

> 如果忘了勾选 `Add Python to PATH`，请把 Python 卸载后重新安装，务必勾上。

### 第二步：下载本项目

方式一（推荐小白）：
1. 打开项目的 GitHub 页面。
2. 点击绿色的 **Code** 按钮 → **Download ZIP**。
3. 下载后解压到一个你熟悉的位置，例如桌面。

方式二：
```bash
git clone <仓库地址>
```

### 第三步：一键打包成 exe

1. 打开解压后的文件夹。
2. 双击 **`一键打包.bat`**。
3. 弹出黑色窗口，自动安装依赖并打包（首次运行大约 3~5 分钟，请耐心等待，不要关掉窗口）。
4. 看到 "打包完成！" 字样即表示成功。

### 第四步：运行

1. 进入项目下的 `dist` 文件夹。
2. 双击 **`订单文件夹工具.exe`** 即可运行。
3. 可以把这个 exe 复制到任何 Windows 电脑上使用（不需要再装 Python）。

### 第五步：首次使用

1. 打开程序后，先在首页设置：
   - **公司资料根目录**：程序会在其下的 `1订单/` 文件夹里创建订单目录结构。
   - **模板文件目录**：你预先准备好的模板文件所在目录（可选，不设置则只创建空文件夹）。
2. 点击对应的 **保存** 按钮。
3. 点击 **🧭 扫描导入业务员** 按钮，程序会自动读取 `1订单/` 下的文件夹，弹出勾选窗口：
   - 勾选属于业务员的文件夹（如「张莹莹」「解小康」）。
   - 对于分公司（如「湖北」），展开后勾选其中的业务员（如「文天堂」）。
   - 不勾选非业务员目录（如「船级证-金山IBC」）。
   - 程序会自动识别业务员下的「进行中订单」「1.进行订单」等中间层，把其中的子文件夹作为客户导入。
4. 然后点「📝 单笔创建」或「📦 批量导入」开始使用。
5. 需要帮助时点击 **❓ 使用帮助**，有完整的图文教程。

## 三、模板文件准备（可选，但强烈推荐）

请按下列目录结构准备 10 个模板文件（扩展名必须一致）：

```
<你选择的"模板文件目录">/
├── 通用/
│   └── CG.xlsx                      （采购合同通用模板）
│
├── 外贸通用/
│   ├── CI.xlsx                      （商业发票）
│   ├── PL.xls                       （装箱单）
│   └── 托书.doc                     （订舱托书）
│
├── 宁夏/                            （产品类别=戊二醛 时使用）
│   ├── 宁夏外贸生产.doc
│   ├── 宁夏外贸发货.docx
│   ├── 宁夏内贸生产.xlsx
│   └── 宁夏内贸发货.xlsx
│
└── 湖北天鹅/                        （产品类别=其他产品 时使用）
    ├── 湖北天鹅外贸生产.xlsx
    └── 湖北天鹅外贸发货.xlsx
```

创建订单时，程序会根据「订单类型 + 产品类别」自动挑选对应模板：

| 订单类型 | 产品类别 | 会复制的模板 |
| --- | --- | --- |
| 外贸 | 戊二醛 | 通用/CG.xlsx + 外贸通用/CI.xlsx + 外贸通用/PL.xls + 外贸通用/托书.doc + 宁夏/宁夏外贸生产.doc + 宁夏/宁夏外贸发货.docx |
| 外贸 | 其他产品 | 通用/CG.xlsx + 外贸通用/CI.xlsx + 外贸通用/PL.xls + 外贸通用/托书.doc + 湖北天鹅/湖北天鹅外贸生产.xlsx + 湖北天鹅/湖北天鹅外贸发货.xlsx |
| 内贸 | 戊二醛 | 通用/CG.xlsx + 宁夏/宁夏内贸生产.xlsx + 宁夏/宁夏内贸发货.xlsx |
| 内贸 | 其他产品 | 通用/CG.xlsx（湖北天鹅暂无内贸模板） |

找不到文件会自动跳过，不影响文件夹创建。

## 四、命名规则自定义

在【模板管理】 → 选模板 → 编辑 → 右侧表格中，可以自由修改每个文件的 `filename` 字段。
支持以下占位符（创建时会自动替换为真实值）：

| 占位符 | 含义 | 示例 |
| --- | --- | --- |
| `<订单号>` | 表单中填写的订单号 | `XS-GAM2508056NH` |
| `<客户名称>` | 表单中填写的客户名称 | `ACME CO., LTD.` |
| `<客户PO号>` | 客户 PO 号 | `PO-12345` |
| `<产品信息>` | 产品信息 | `戊二醛 200KG` |
| `<业务员>` | 当前选择的业务员姓名 | `张三` |
| `<日期>` | 创建当天日期 YYYYMMDD | `20250821` |
| `<SHXY编号>` | 备用占位 | `SHXY` |

示例：

- `CI-<订单号>.xlsx` → `CI-XS-GAM2508056NH.xlsx`
- `销售合同-<客户名称>-<日期>.pdf` → `销售合同-ACME-20250821.pdf`

此外，`file_template` 字段支持 `[产地]` 标记：

- `[产地]外贸生产` 在产品类别=戊二醛 时变成 `宁夏/宁夏外贸生产.doc`
- `[产地]外贸发货` 在产品类别=其他产品 时变成 `湖北天鹅/湖北天鹅外贸发货.xlsx`

## 五、数据存放位置

所有数据以 JSON 形式保存在你选择的 **根目录** 下的 `.order_tool/` 子目录中：

```
<根目录>/.order_tool/
├── config.json            程序配置（模板目录、上次选择等）
├── salespersons.json      业务员和客户列表（含 rel_path / mid_layer）
├── history.json           历史记录
└── templates/             模板文件夹
    ├── standard_export.json
    ├── standard_domestic.json
    ├── <业务员>_default_export.json      业务员个人外贸模板
    ├── <业务员>_default_domestic.json    业务员个人内贸模板
    └── <业务员>_<客户>_<type>.json       客户专属模板
```

### salespersons.json 新结构（v2）

```json
{
  "list": [
    {
      "name": "张莹莹",
      "rel_path": "张莹莹",
      "mid_layer": "",
      "customers": ["LLC KEMIKLKRAFT", "客户B"]
    },
    {
      "name": "文天堂",
      "rel_path": "湖北/文天堂",
      "mid_layer": "",
      "customers": ["客户A", "客户B"]
    },
    {
      "name": "解小康",
      "rel_path": "解小康",
      "mid_layer": "进行中订单",
      "customers": ["Arxada Tray Siam", "BASF Nederland"]
    }
  ]
}
```

- `rel_path`：业务员目录相对于 `<根目录>/1订单/` 的路径，支持「湖北/文天堂」这种分公司写法。
- `mid_layer`：中间层文件夹名，例如「进行中订单」「1.进行订单」。留空表示没有中间层。
- 旧版（只有 `name` 和 `customers`）会在首次读取时自动补全缺失字段。

> 想在多台电脑上共用，可以把根目录放到共享盘或云盘里。

## 六、常见问题

**Q：首次打包时提示 pip 太慢？**
一键打包脚本已经切到清华源，如果仍然慢，可以使用手机热点再试一次。

**Q：双击 exe 没反应？**
用记事本打开 `dist/订单文件夹工具.exe.log`（如果有），或直接用命令行运行查看错误。

**Q：怎么备份数据？**
直接把 `<根目录>/.order_tool/` 整个文件夹复制到其它地方即可。

**Q：可以修改公司标准模板吗？**
可以在【模板管理】中"编辑"，但会强制你另存为"个人模板"或"客户模板"，标准模板文件本身不会被覆盖。
